# サムネイル画像仕様 (Thumbnail Specifications)

本ドキュメントは、Preludio Labプロジェクトにおけるコンテンツ用サムネイル画像の標準仕様を定義します。
今後、画像素材を作成または生成AI（DALL-E 3, Midjourney等）で生成する際は、本仕様をコンテキストとして使用してください。

## 1. 画像規格

Webサイトのパフォーマンスと、Retinaディスプレイ（高解像度端末）での表示品質を両立させるための最適設定です。

| 項目 | 仕様 | 備考 |
| :--- | :--- | :--- |
| **基本解像度** | **幅 1600px** | **「大は小を兼ねる」戦略**を採用。ヒーローイメージ（詳細ページ）とサムネイル（一覧ページ）で同一のマスター画像を使用する推奨サイズ。 |
| **アスペクト比** | **16:9** | 標準的なサイズは **1600x900px**。 |
| **ファイル形式** | **WebP (.webp)** | 原則として**WebPを推奨**。マスターデータとして高画質・高圧縮で保存する。 |
| **ファイル容量** | **500KB以下** | Next.jsが配信時に最適化するため、マスター画像としては多少大きくても許容されるが、処理負荷軽減のためメガバイト単位は避ける。 |
| **ファイル名規則** | ケバブケース | 例: `beethoven-symphony-5.webp` |

## 2. デザイン・生成ガイドライン

AIに画像生成を指示する際のガイドラインです。

*   **スタイル**: モダン、抽象的、エレガント (Modern, Abstract, Elegant)
*   **テキスト**: **画像内に文字を含めないこと**。
    *   タイトルやカテゴリ名は、Webサイト上でHTML/CSSとしてオーバーレイ表示されます。画像内に文字焼き込みを行うと、翻訳やレスポンシブ対応の妨げになります。
*   **構図**: 中央に重要な要素を配置し、四隅はトリミングされても問題ないようにする。

## 3. 実装上の取り扱い (エンジニア向け)

フロントエンド実装においては、画像の最適化機能を最大限に活用してください。

*   **コンポーネント**: 必ず Next.js の `next/image` (`<Image />`) コンポーネントを使用する。
    *   `src`: 上記仕様の **1600px WebPマスター画像** を指定。
    *   `sizes`: **必須**。`sizes="(max-width: 768px) 100vw, 33vw"` のように、ブラウザが「今どのサイズで表示すべきか」を判断できる情報を記述する。
    *   **理由**: Next.jsのImage Optimization機能により、サーバー側で自動的に最適なサイズへリサイズして配信されるため。`sizes`属性がないと、サムネイル表示時でもフルサイズの画像がダウンロードされる可能性がある。

## 4. 戦略: 「大は小を兼ねる」 (One Master Strategy)

Next.js環境においては、サムネイル用(小)とヒーロー用(大)を分けて管理する必要はありません。

*   **運用**: **1600px〜1920px (WebP)** の高画質なマスター画像を1つだけ用意する。
*   **配信**: Next.jsがアクセス端末や表示領域に合わせて、そこから自動的に小サイズ（400px, 800pxなど）を生成・キャッシュして配信する。
*   **メリット**:
    *   **一貫性**: 画像管理が1コンテンツ1枚で済み、運用ミスが減る。
    *   **適応性**: 将来より高解像度なデバイスが出ても、マスターが高画質であればコード修正だけで対応可能。

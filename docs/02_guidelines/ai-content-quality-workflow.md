# PreludioLab 記事品質担保・戦略的AIワークフロー報告書

## 1. はじめに

本報告書は、個人開発でありながら10,000記事以上のエンタープライズ品質を実現するための、生成AIを活用した自律的品質管理システムについてまとめたものである。本システムは、ユーザーの「没入感」と「信頼性」を最優先しつつ、持続可能な収益性を両立させることを目的とする。

---

## 2. 品質評価の多角的フレームワーク

記事の検品を行うAIエージェントには、直交性と網羅性を担保した**「6＋3」の評価軸**を導入する。評価は一貫性を保つため5段階スコアで行う。

### A. コンテンツ品質の6軸（Content Critic）

| 評価軸                 | 定義                                         | 役割                         |
| ---------------------- | -------------------------------------------- | ---------------------------- |
| **Fact (正確性)**      | 時代背景、作品番号、調性、事実関係の正確さ   | 信頼性の担保（必須）         |
| **Insight (専門性)**   | 楽曲形式や語法に対する深い音楽的分析         | 熟練者の満足度向上           |
| **Narrative (親密性)** | 執筆者の情熱や曲への愛情、ワクワクする語り口 | 初心者の没入感とブランド構築 |
| **UX Sync (体験性)**   | 譜例、音源、テキスト解説の三位一体の整合性   | サイト体験の核心             |
| **i18n Readiness**     | 自動翻訳に耐えうる論理構造と文化的配慮       | グローバル展開の効率化       |
| **Compliance**         | 著作権保護期間の遵守、不適切表現の排除       | 法的リスクの回避（必須）     |

### B. マネタイズ品質の3軸（Monetize Critic）

| 評価軸                 | 定義                                           | 役割                        |
| ---------------------- | ---------------------------------------------- | --------------------------- |
| **Contextual Linkage** | 解説の流れに沿った自然な誘導ができているか     | コンバージョン率（CVR）向上 |
| **Product Relevance**  | 紹介楽曲とリンク先商品の整合性があるか         | ユーザーの利便性向上        |
| **Visual Harmony**     | 広告が没入体験（譜例や動画）を阻害していないか | ユーザーロイヤリティの維持  |

## 3. 戦略的ティアリングによるリソース最適化

すべての記事に一律の基準を課すのではなく、ビジネスインパクトに応じた2つの戦略を動的に適用する。

### 戦略(A)：集客・信頼獲得型（Flagship Articles）

- **対象:** 有名曲、検索ボリュームの大きい楽曲。
- **目標:** サイトの権威向上と新規ユーザー獲得。
- **基準:** Content Criticスコア **平均4.5以上** を必須とし、マネタイズ導線は最小限（2.0以上）に留める。

### 戦略(B)：収益実現型（Conversion Articles）

- **対象:** 購買意欲の高いファン層、楽器奏者向けの楽曲。
- **目標:** 収益（LTV）の最大化。
- **基準:** Content Criticスコア **平均3.5以上** を維持しつつ、Monetize Criticスコア **4.0以上** を必須とする。

---

## 4. 自律型推論ループのワークフロー

以下のパイプラインにより、人間の介入を最小限に抑えつつ品質を制御する。

1. **Planning Phase:** Tiering Agentが楽曲の市場性を判定し、戦略（AまたはB）を決定。
2. **Drafting Phase:** Draft AgentがMDX形式で記事草案を作成。
3. **Content Loop:** Content Criticが6軸で採点。不合格（FAIL）なら具体的なAction Itemを提示し、Draft Agentが修正。合格するまで繰り返す。
4. **Monetize Phase:** Monetize Agentが戦略に基づき導線を挿入。
5. **Monetize Loop:** Monetize Criticが3軸で採点。没入感を損なう、または導線が弱い場合は修正指示を出し、合格するまで繰り返す。
6. **Final Polish:** 最終的なトーン＆マナーの微調整を行い、デプロイ。

---

## 5. 今後の展望と拡張性

- **動的な基準調整:** ユーザーの反応（PV/CVR/滞在時間）をAIが学習し、評価軸の重み付けを自動更新するフィードバックループの構築。
- **オリジナルコンテンツとの連動:** MusicXMLから自動生成したアナリーゼ譜（PreludioLab Edition）の販売導線を、Monetize Agentの最優先項目として組み込む。
